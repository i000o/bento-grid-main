## Process

- I worked mobile-first on this project after having gone back and forth in the past. I can see that a mobile-first flow is more conducive to a straightforward design flow. Another dev explained it to me as the building blocks and then a rearrangement of them for desktop. Since then, it's worked well for me to do it this way.
- The mobile design was easy to follow as a single column flex flow. I enjoy the details of design in each card and can program them quite swiftly.
- Once I pivoted to desktop, things slowed down. I was unfamiliar with responsive design using tailwind but once I learnt the breakpoints, this was simpler and I do see why devs use this language over CSS or Sass. I'd like to continue using it in the future.
- I had initial struggles with how my Grid should be setup. I made `<body>` the Grid, which I now know that often, you'd include a container for it but I don't think it matters much in this instance. I'll remember this in the future. A big confusion I had was to include the gaps in my consideration, which I didn't need to. Eventually, I found a layout, but judging by the design, I feel that it's not exact, but that's okay for now.
- I do enjoy using Grid when I do, but I don't use it so often as I so reliably use Flex, but I'd like to take on more challenging layouts now where I can practice it more, particularly with Tailwind under my belt.
- I made all the cards flex containers within the Grid, which I feel worked well.
- I don't struggle with matching the designs well. Maybe in future, I can opt for the platform's new Product challenges over Design ones so I can make more of those decisions myself, but for now, I'm usually prioritising a technical skill rather than experimenting with UI personally.

## Lessons

1. I tried to make good use of global styles in Tailwind. I do enjoy using it as I like to keep my HTML clear of anything that doesn't need to be there. However, I found later that I could've done this with the `tracking` property, but I didn't anticipate repeating it as much as I did. I can change this at any time, though.
2. I need more Grid practice! I'm rusty! I'm so comfortable with Flex, and since pivoting to Tailwind from Sass, I'm unfamiliar, but will try to reestablish that muscle memory again soon.
3. I'm great at replicating design. It's not difficult for me. I should maybe try some Product challenges next.
4. I like Tailwind, and take pride in keeping it as uncluttered as possible. It gives me confidence in my CSS property targetting because I don't waste lines on things that don't make impactful change.
5. My confidence in Tailwind is growing. I'm better at remembering syntax now and am confident finding it in the documentation if I need to. I want to use it more.
